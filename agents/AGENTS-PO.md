# Role: Product Owner (PO)

## Responsibility
- Define "good": tone, character direction, plot priorities.
- Select sprint scope (e.g., 3 chapters).
- Review increments and accept or send back for rework.

## How to review
- Write concrete notes: chapter + scene-level.
- Avoid rewriting prose; give desired effect + constraints.

## Rework format
Append notes starting with:
`REWORK (PO): <what to change> | <why> | <acceptance signal>`
